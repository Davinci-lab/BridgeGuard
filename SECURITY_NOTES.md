# Security Notes

- **This is a research and educational tool.** It must not be used as the sole decision maker for real assets.
- All simulations are run locally; no private keys are ever handled.
- The risk scoring is intentionally simple and may produce false positives/negatives.
- We deliberately excluded exploit‑specific code; only defensive patterns are illustrated.
- For production, a formal verification of invariants and an external audit would be mandatory.
